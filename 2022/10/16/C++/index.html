C++中的Class相关知识

  • inline（内联）函数

    ​ 函数若在class body内定义完成，便自动成为（inline）内联候选

  • access level 访问级别

    • public
    • private
    • protect

  • 构造函数

    • 构造函数的变量设值有两阶段：1、初始化，2、赋值。如果在body内设值，则浪费了初始化阶段的时间，效率低下，

    • 构造函数可以有很多个重载（overloading）

    • 构造函数放在private中——单例模式（Singleton），外界不能创建其对象，通过静态方法创建。

  • 常量成员函数（const member functions）

    const修饰的函数，不对属性变量做修改。const修饰的对象只能调用const修饰的函数。如果const修饰的变量调用非const修饰的函数，函数中可能存在修改属性变量的行为，这样就会导致程序错误。

  • **参数传递 ** pass by value VS. pass by reference(to const)

    • 前加const是为了避免在函数内通过引用修改数据，因为传递过去的是引用，一旦修改可能导致其他地方出现问题。

    • 前不加const就说明会在函数总修改数据。

    • 什么情况下可以pass by reference

      • 局部变量不可以传递引用

    • 什么情况下可以pass by value

      • 都可以

  • 返回值传递

    • 尽量用引用(reference)做返回值传递，提高效率。

    • return by reference的语法：传递者无需知道接收者是以reference形式接收

栈(stack)与内存管理

  • 栈

    • 栈（stack），是存在于某作用域的一段内存空间。例如当你调用函数，函数本身即会形成一个stack用来放置它所接收的参数，以及返回地址。

      • 在函数本身内声明的任何变量，其所使用的内存块都取自上述stack。

    • 堆（Heap），是指操作系统提供的一段global内存空间，程序可动态分配其中的若干区块。

    • stack objects的生命期

      • c1就是所谓的stack object，其生命在作用域结束之际结束，这种作用域内的object，又称为auto object，因为他会被自动清理。

    • static local objects的生命期

      • c2便是所谓的static object，其生命在作用域结束之后仍然存在，直到整个程序结束。

    • global objects的生命期

      • 上面的c3便是所谓的global object,其生命在整个程序结束之后才结束，你可以把他视为一种static object，其作用域是整个程序。

    • heap object的生命期

      • p所指的便是heap object，其生命在它被deleted之际结束
      • p2出现内存泄漏，因为当p2所在作用域结束之际p2死亡，但是p2所指的heap object（new出来的都是heap object）仍然存在，在作用域职位再也看不到p2了，也就没机会delete p2了。

组合与继承

  • 继承

    • 表示is-a的关系，（C++的继承语法有三种）

    • 继承关系下的构造和析构

      • 构造由内而外

        ​ Derived的构造函数首先调用Base的默认构造函数，然后才执行自己的构造函数

      • 析构由外而内

        ​ Derived的析构函数首先执行自己的析构函数，然后才调用Base的析构函数

  • 继承与虚函数（Template Method设计模式）

    • non-virtual函数：你不希望子类（derived class）重新定义它。

    • virtual函数：你希望子类（derived class）重新定义它，且拥有默认定义

    • pure virtual函数：你希望子类（derived class）一定要重新定义它，你对他没有默认定义

    ​ 案例如下：Template Method设计模式：将其中的Serialize函数设计为虚函数，交给子类去定义。但是OnFileOpen函数却是非虚函数，中间却引用了Serialize函数，所以只要子类重新定义了Serialize函数，OnFileOpen函数所做的功能就会随之改变。

  • 复合/组合

    • 表示has-a的关系（软件工程理论中的组合关系），某个类中有另外一个类的对象。

    • 复合关系下的构造和析构

      • 构造由内而外

        ​ Container的构造函数首先调用Component的构造函数，然后再执行自己的默认构造函数

      • 析构由外而内

        ​ Container的析构函数首先执行自己，然后再执行Component的析构函数

  • 委托（Composition by reference）

    • 委托关系类似于软件工程理论中的聚集关系,通过指针指向另外一个类的对象（不提前创建该类对象，可以后期再创建，通过指针指向）

  • 继承与复合关系下的构造与析构

    ​

    ​ 在这种继承与复合关系下，构造函数是Base类先执行，其次是Component类执行，最后是Derived执行。

    ​ 析构函数的执行顺序：由外而内，Derived先执行，Base执行，Component最后执行。

  • 委托加继承关系（观察者模式）

    • 观察者模式：类subject中有一个Observer对象集合，而Observer的子类对象都可以替换父类Observer对象，进而写入到这个集合中。而Observer的子类可以重写其父类Observer的方法，以实现额外的功能
